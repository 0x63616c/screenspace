# Swift Features & Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a type-safe Swift API client from the OpenAPI spec, extract all business logic into testable ViewModels, apply a full accessibility pass, fix all known deprecations, standardize UI components, and write Swift Testing suites for every ViewModel.

**Architecture:** ViewModels are `@Observable @MainActor` classes that take protocol dependencies via init (never `AppState` directly). Views own a ViewModel instance or receive one via environment injection. The Swift API client is generated by `swift-openapi-generator` as a SwiftPM build plugin, replacing hand-written `APIModels.swift`. Tests use `MockNetwork`, `MockEventLog`, `MockKeychain`, `MockFileSystem`, and `MockConfigStore` from Plan 4's platform layer.

**Tech Stack:** swift-openapi-generator 1.x (build plugin), swift-openapi-runtime 1.x, Swift Testing (`@Test`/`@Suite`), SwiftUI, `@Observable`, macOS 15+

**Depends on:** Plan 1 (Go Foundation — `api/openapi.yaml` exists and is complete), Plan 4 (Swift Foundation — platform protocols, Live/Mock implementations, EventLog, typed enums, `@Observable` migration complete)

## Errata (post-review fixes, apply throughout)

1. **Execution order: `appState.cache` (E2.11) must be added before E2.1.2** which references it. Move E2.11 (AppState composition root wiring) to run BEFORE E2.1. Or: add `cache: CacheProviding` to AppState as part of E2.1 setup.
2. **Remove `fatalError` from `authHeader()` stub.** Either delete the method entirely or have it return an empty string. Agents will copy the `fatalError` verbatim and it will crash at runtime.
3. **Fix `UploadViewModel` event name:** `"wallpaper_downloaded"` on upload success is wrong. Use `"wallpaper_uploaded"` instead.
4. **`NetworkProviding` vs `APIProviding`:** The spec defines `NetworkProviding` as raw transport (`data(for:) async throws -> (Data, URLResponse)`). This plan redefines it as a typed domain API. To avoid spec contradiction, introduce a NEW `APIProviding` protocol for the typed domain methods. Keep `NetworkProviding` as-is from Plan 4 (raw transport). ViewModels should depend on `APIProviding`, not `NetworkProviding`. The `APIService` wrapper from E1.5 implements `APIProviding`.
5. **Add E3 step for "Now Playing" menu item.** Add `.accessibilityLabel("Now playing: \(title)")` and mark as a live region with `.accessibilityAddTraits(.updatesFrequently)` so VoiceOver announces changes.
6. **E3.3 "verify it's complete" is vague.** Replace with: check that the drop zone has `.accessibilityLabel("Drop video file here")` and `.accessibilityHint("Imports the video into your library")`.

@MainActor

    func login(email: String, password: String) async throws -> AuthToken {
        let response = try await client.loginUser(.init(body: .json(.init(email: email, password: password))))
        switch response {
        case .ok(let ok):
            let body = try ok.body.json
            try keychain.save(key: KeychainKey.token, data: Data(body.token.utf8))
            return AuthToken(token: body.token, role: UserRole(rawValue: body.role) ?? .user)
        case .undocumented(let status, _):
            throw APIError.httpError(status: status)
        }

> **Note on auth middleware:** swift-openapi-generator supports `ClientMiddleware` for injecting auth headers. Create `AuthorizationMiddleware: ClientMiddleware` that reads the token from `KeychainProviding` and injects `Authorization: Bearer <token>` on every request. - [ ] **E1.6** — Create `app/Sources/ScreenSpace/Core/AuthorizationMiddleware.swift`:

```swift
import Foundation
import OpenAPIRuntime
import HTTPTypes

struct AuthorizationMiddleware: ClientMiddleware {
    let keychain: KeychainProviding

    func intercept(
        _ request: HTTPRequest,
        body: HTTPBody?,
        baseURL: URL,
        operationID: String,
        next: (HTTPRequest, HTTPBody?, URL) async throws -> (HTTPResponse, HTTPBody?)
    ) async throws -> (HTTPResponse, HTTPBody?) {
        var request = request
        if let tokenData = keychain.load(key: KeychainKey.token),
           let token = String(data: tokenData, encoding: .utf8) {
            request.headerFields[.authorization] = "Bearer \(token)"
        }
        return try await next(request, body, baseURL)
    }
}
```

import Foundation

@Observable
@MainActor
final class HomeViewModel {
    private let api: NetworkProviding
    private let eventLog: EventLogging

    var popular: [WallpaperCardData] = []
    var recent: [WallpaperCardData] = []
    var featured: WallpaperCardData?
    var isLoading = true
    var error: String?
    var selectedDetail: WallpaperDetail?

    init(api: NetworkProviding, eventLog: EventLogging) {
        self.api = api
        self.eventLog = eventLog
    }

    func load() async {
        guard !Task.isCancelled else { return }
        isLoading = true
        error = nil
        do {
            let pop = try await api.popularWallpapers(limit: 10, offset: 0)
            guard !Task.isCancelled else { return }
            let rec = try await api.recentWallpapers(limit: 10, offset: 0)
            popular = pop.items
            recent = rec.items
            featured = popular.first
            eventLog.log("wallpapers_loaded", data: ["popular": popular.count, "recent": recent.count])
        } catch {
            self.error = "Community gallery unavailable. Connect to a server in Settings."
            eventLog.log("error", data: ["context": "home_load", "message": error.localizedDescription])
        }
        isLoading = false
    }

    func fetchDetail(id: String) async {
        do {
            selectedDetail = try await api.getWallpaper(id: id)
        } catch {
            self.error = "Failed to load wallpaper details."
        }
    }

    func clearDetail() {
        selectedDetail = nil
    }
}
